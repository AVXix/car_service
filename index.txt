Index module reference (`index.php`)
====================================

1) File structure (index side)
------------------------------
- `index.php`
  - Loads `config.php`, starts session, runs `handlers/index_page_request_handler.php`.
  - Chooses one view:
    - `views/index_authentication_section.php` when user is not logged in.
    - `views/index_appointment_section.php` when user is logged in.
  - Loads `js/index-page-interactions.js`.

- `handlers/index_page_request_handler.php`
  - Main index request orchestrator.
  - Initializes all shared variables for auth + booking views.
  - Loads:
    - `handlers/index_auth_request_handler.php`
    - `handlers/index_booking_request_handler.php`.

- `handlers/index_auth_request_handler.php`
  - Handles user auth actions:
    - `create_account`
    - `sign_in`
    - `logout_user`.

- `handlers/index_booking_request_handler.php`
  - Handles booking actions and booking data loading:
    - load mechanics list,
    - load current user's appointment list,
    - process `request_appointment`,
    - process `cancel_appointment`.

- `views/index_authentication_section.php`
  - Renders user sign-in or create-account form.
  - Shows auth errors/notifications.

- `views/index_appointment_section.php`
  - Renders logged-in booking UI:
    - account header,
    - current appointment cards,
    - cancel appointment action,
    - new appointment modal form.

- `js/index-page-interactions.js`
  - Frontend behavior:
    - autofocus username field,
    - enforce minimum date for appointment input,
    - modal open/close handling,
    - logout confirmation,
    - prevent accidental double form submit.


2) How implementation works (index request flow)
------------------------------------------------
Step 1: Entry
- Browser opens `index.php`.
- Common setup runs (`config.php`, session start, index page handler).

Step 2: Handler orchestration
- `index_page_request_handler.php` prepares shared variables.
- Auth handler processes auth-related actions.
- Booking handler processes booking-related actions and data loading.

Step 3: Action processing
- Auth handler handles create/sign-in/logout requests.
- Booking handler validates booking form requests and cancellation requests.

Step 4: Data loading
- Booking handler loads mechanics for select options.
- Booking handler loads appointments that belong to current user.

Step 5: Rendering
- Not authenticated -> authentication view.
- Authenticated -> appointment/booking view.
